1. Layer Architecture

The Kinana Platform follows a distributed microservices architecture organized into 6 distinct layers. Each layer serves a specific purpose in the overall system architecture, from handling external traffic to managing persistent storage.

2. Microservices Overview

The platform consists of 20+ microservices organized into functional groups: Core Services, Application Services, PDF Processing, LTI Integration, Data Layer, and Storage Layer. Each service is independently deployable and scalable.

3. Service Communication Patterns

Services communicate using Kubernetes internal DNS for service discovery. The Redis cache is shared across all core services for session management and performance optimization. Databases are accessed internally (MySQL) and externally (SQL Server for LTI).
